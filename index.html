Unity代码剥离相关

发表于 2024-06-07 分类于 Unity

Unity的代码剥离

如同编译Java时需先将源码转成字节码，Unity打包编译源码时也有类似过程。

在此过程中，Unity会检测代码引用情况，将类中从未使用过的成员、方法和不会执行到的代码删除，此过程称为代码剥离。这样做的效果是能够显著减少包体大小。

这个检测过程是静态的，动态生成代码中的引用情况无法检测到。所以例如反射的情况，检测时未发现相关引用会将其删除，导致反射时找不到类或方法。

Unity移动端异形屏相关

发表于 2024-01-30 分类于 Unity

异形屏

异形屏指非传统矩形的屏幕。一些常见屏幕异形的原因：

  • 刘海屏，多是屏幕上方增加凹槽或打孔放置前置摄像头；
  • 曲面屏，多是屏幕沿左右两边弯曲。
  • iPhone的虚拟Home Indicator，屏幕下方的小横线按钮。



Unity做移动端开发时，UI画布、各种元素都是基于矩形的，默认不考虑异形屏，直接铺满屏幕，但某些情况下，屏幕的“异形”部分会挡住应用内的关键显示，需要做相应处理。

DOTween停止动画对回调的影响

发表于 2023-11-28 分类于 Unity

DOTween停止动画对回调的影响

DOTween中的回调，一般有OnComplete加在动画结束，以及Insert/AppendCallback插在动画中间。OnStart也可添加回调，但停止动画时一般都已经执行，故不做讨论。

DOTween的停止动画，可用Complete方法使动画立即播放完毕，也可用Kill方法使动画中断。

以下是几种DOTween停止动画时，对回调方法执行情况影响的测试结果：

  • DOTween.Complete(false) 方法会执行OnComplete回调，但不会执行AppendCallback中的方法
  • DOTween.Complete(True) 方法会执行OnComplete回调，也会执行AppendCallback中的方法
  • DOTween.Kill(false) 暴力停止，动画若未执行完，则其主体会留在当时所处状态。后面未执行到的任何回调也都不再执行
  • DOTween.Kill(True) 方法会执行OnComplete回调，但不会执行AppendCallback中的方法
