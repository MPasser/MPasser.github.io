Redis基础

发表于 2022-08-21 分类于 Database ， Redis

概述

Redis是一个键值数据库。没有表、库或约束，是一种 NoSQL 数据库（非关系型的数据库）。

NoSQL相对于SQL

SQL NoSQL
数据结构 结构化 非结构化
数据关联 关联的 无关联的
查询方式 SQL查询 非SQL
事务特性 ACID BASE
存储方式 磁盘 内存
扩展性 垂直 水平
适用场景 数据结构固定、数据安全性、一致性要求较高 数据结构不固定、性能要求高
  • 结构化。SQL数据库（关系型数据库）的数据是结构化的，有主键约束、唯一性类型的约束等；NoSQL是非结构化的，约束相对不严格，数据大致有键值类型（Redis）、文档类型（MongoDB）、列类型（HBase）、Graph类型（Neo4j）几种组织形式。
  • 关联性。SQL数据库的表之间可能通过外键相关联；NoSQL想做到同效果只能另外编写业务逻辑实现，数据库不负责维护。
  • SQL查询。SQL数据库使用SQL语言查询；NoSQL没有统一标准，例如Redis 用命令 get user:1，MongoDB 用函数 db.user.find({_id:1})，elasticsearch 用请求 GET http://localhost:9200/users/1
  • 事务。SQL有事务属性；NoSQL无事务，牺牲强一致性，来获得可用性。
  • 扩展性。NoSQL利用哈希将数据拆分到不同节点，在分布式系统扩展节点会很方便。

Redis简述

始于2009，Redis（Remote Dictionary Server）远程词典服务器，是一个基于内存的键值型NoSQL数据库。

有这些特征：

  • 键值（key-value）型，value支持多种数据结构，功能丰富。
  • 单线程，每个命令具备原子性。
  • 低延迟，速度快（基于内存、IO多路复用、良好的编码）。
  • 支持数据持久化。
  • 支持主从集群、分片集群。

配置项 含义
bind 监听地址，默认只监听本机；设为0.0.0.0，则可在任意IP访问服务
port 监听端口。默认6379

requirepass 若设置，则访问Redis服务必须输入密码

database 数据库数量。默认有16个库，编号0~15
maxmemory 设置Redis能够使用的最大内存
logfile 日志文件

Redis命令

Redis是一个键值数据库，key一般是String类型，但value有多种类型

类型 含义 举例
String 字符串 Hello World
Hash 哈希表 {name:”Adam”,age:16}
List 链表 [A->B->C->C]
Set 集合 {A,B,C}
SortedSet 有序集合 {A:1,B:2,C:3}

以上是value的基本类型，以下是一些特殊类型

类型 含义 举例
GEO 地理坐标 {A:(120.3,30.5)}
BitMap 位图 0110110101110101011
HyperLog 去重统计算法 0110110101110101011

操作每种数据类型使用的命令是不同的。

通用命令

命令 参数 含义
KEYS pattern 返回匹配模式的所有key名称，pattern中使用的通配符见表下说明
DEL key … 删除指定key
DEL key … 判断key是否存在，存在返回1.反之返回0
EXPIRE key seconds 给一个key设置有效期，过了有效期时间后，key被自动删除
TTL key 查询一个key剩余的有效期，永久有效key的值为-1，已过期（不存在）的key值为-2

KEYS命令有使用模糊查询，数据量大时会很慢，阻塞服务，不建议在生产环境下使用。

向redis中存入key时建议都设置一个有效期。

Redis 使用的通配符如下：

  • ?表示单个任意字符。h?llo matches hello, hallo and hxllo
  • *表示任意个任意字符。h*llo matches hllo and heeeello
  • []表示在匹配括号中的一个字符。h[ae]llo matches hello and hallo, but not hillo
  • ^表示非。h[^e]llo matches hallo, hbllo, … but not hello
  • -表示字符范围。h[a-b]llo matches hallo and hbllo

String类型命令

根据字符串格式不同，String类型数据又可以分成3类：

  • String：普通字符串。如 Hello World
  • int：整型。如 10
  • float：浮点型。如 92.5

其中数字类型可做自增自减操作。

无论哪种格式，底层都是字节数组形式存储，只是编码方式不同，字符串类型的最大空间不能超过512M。

命令 参数 含义
SET key value 新增或修改键值对
GET key 获取key对应的value
MSET key value … 批量设置键值对
MGET key … 批量获取value
INCR key 让一个整型数据自增1
INCRBY key increment 让一个整型数据自增，自定义步长
INCRBYFLOAT key increment 让一个浮点型数据自增，自定义步长
SETNX key value 新增键值对，若键值对存在则不执行
SETEX key seconds value 设置一个键值对并设置有效期

Hash类型命令

Hash类型，也叫散列，其value是一个无序字典，类似于Java中HashMap结构。

String结构是将对象序列化为JSON字符串后存储，当需要修改对象某个字段时很不方便；Hash结构可以将对象中的每个字段独立存储，可以针对单个字段做CURD。

常用命令有：

命令 参数 含义
HSET key field value 添加或修改hash类型key的field的值
HGET key field 获取指定key的field的值
HMSET key field value [field value …] 批量添加field的值
HMGET key field 批量获取多个field值
HGETALL key 获取所有的field和value
HKEYS key 获取所有的field
HVALS key 获取所有的value
HINCRBY key field increment 让一个hash类型key的字段值自增，自定义步长
HSETNX key field value 添加一个field的值，如果field已存在则不执行

List类型命令

Redis中的List类型与Java中的LinkedList类似，可看做一个双向链表结构，支持双向检索。有这些特征：

  • 有序
  • 元素可重复
  • 插入和删除快
  • 查询速度不快

常用命令有：

命令 参数 含义
LPUSH key value [value …] 向List左侧插入一个或多个元素
LPOP key 移除并返回List左侧的第一个元素，List为空则返回nil
RPUSH key value 向List右侧插入一个或多个元素
RPOP key 移除并返回List右侧的第一个元素
LRANGE key start stop 返回一段索引范围内的所有元素
BLPOP key … timeout 执行LPOP，如果List为空则等待指定时间，并不直接返回nil
BRPOP key … timeout 执行RPOP，如果List为空则等待指定时间，并不直接返回nil

List的索引从0开始，从左到右；

BLPOP/BRPOP效果类似于阻塞队列。

Set类型命令

Redis的Set结构与Java中的HashSet类似，因为也是一个Hash表，因此具备与HashSet类似的特征：

  • 无序
  • 元素不可重复
  • 查询速度快
  • 支持交集、并集和差集等功能

常用命令有：

命令 参数 含义
SADD key member … 添加一个或多个元素
SREM key member … 删除指定元素
SCARD key 返回元素的个数
SISMEMBER key member 判断member元素是否在Set中存在
SMEMBERS key 获取所有元素
SINTER key … 取指定Set的交集
SDIFF key … 取指定Set的差集
SUNION key … 取指定Set的并集

SortedSet类型命令

Redis的SortedSet是一个有序Set集合。其中每个元素都有一个score属性，可基于score属性对元素排序，底层的实现是一个跳表（SkipList）加Hash表。有以下特征：

  • 可排序
  • 元素不可重复
  • 查询速度快

因为SortedSet的可排序特性，常被用来实现排行榜这样的功能。

常用命令有：

命令 参数 含义
ZADD key score member [score member …] 添加一个或多个元素
ZREM key member … 删除指定元素
ZCARD key 返回元素的个数
ZSCORE key member 获取指定元素的score值
ZRANK key member 获取指定元素的排名
ZCOUNT key 获取元素个数
ZINCRBY key increment member 让指定元素自增，自定义步长
ZRANGE key start stop 按score排序后，获取指定排名范围内的元素
ZRANGEBYSCORE key start stop 按score排序后，获取指定排score值范围内的元素
ZINTER/ZDIFF/ZUNION key … 取指定Set的交集/差集/并集

所有的排序默认都是升序，如果需要降序，则在命令中的Z后添加REV即可，例如ZREVRANK、ZREVRANGE。

Java客户端

  • Jedis。以Redis命令作为方法名称，学习成本低，简单实用。但Jedis实例是线程不安全的，多线程环境下需要基于连接池来使用。
  • Lettuce。基于Netty实现，支持同步、异步和响应式编程，且线程安全。支持Redis的哨兵模式、集群模式和管道模式。
  • Redissio是一个基于Redis实现的分布式、可伸缩的Java数据结构集合。包含了诸如Map、Queue、Lock、Semaphore和AtomicLong等强大功能。
  • Spring Data Redis，底层实现的API兼容Jedis和Lettuce。

Jedis连接池

Jedis本身是线程不安全的，并且频繁地创建和销毁连接会有性能损耗，因此推荐使用Jedis连接池。

SpringDataRedis

SpringData是Spring中数据操作的模块，包括对各种数据库的集成，其中对Redis的集成模块就叫做SpringDataRedis。

  • 提供了对不同Redis客户端的整合（Jedis和Lettuce）
  • 提供了RedisTemplate统一API来操作Redis
  • 支持Redis的发布订阅模型
  • 支持Redis哨兵和Redis集群
  • 支持Lettuce的响应式编程
  • 支持基于JDK、JSON、字符串和Spring对象数据的序列化及反序列化
  • 支持基于Redis的JDKCollection实现

SpringData中提供了RedisTemplate工具类，封装了各种对Redis的操作。并将不同数据类型的操作API封装到了不同的类型中：

API 返回值类型 说明
redisTemplate.opForValue ValueOperations 操作String类型数据
redisTemplate.opForHash HashOperations 操作Hash类型数据
redisTemplate.opForList ListOperations 操作List类型数据
redisTemplate.opForSet SetOperations 操作Set类型数据
redisTemplate.opForZSet ZSetOperations 操作SortedSet类型数据
