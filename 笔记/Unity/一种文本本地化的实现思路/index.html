一种文本本地化的实现思路

发表于 2023-01-10 分类于 Unity

前言

本地化，就是工程中的文本能支持多种语言，能根据系统信息或手动设置自动变成不同语言。

在网上找了好些方案，找到了一种感觉最简单的，但又觉得有点憨，维护成本有点高的。但总之还能用，就简单做下记录，高级方案以后再看看吧。

游戏引擎使用Unity，语言使用C#

思路

本地化文件用json格式保存，不同语言使用各自的json文件，数据格式就是一个列表，每个元素包含一个键值对。

对于同一个Text文本，不同json文件中的键是相同的，值就是各个语言的本地化文本。

几个核心的部件如下：

pojo类

  • 本地化文件用json格式保存，用UnityEngine自带的JsonUtility做序列化，所以要有一个pojo类存数据。

本地化器：

  • 每个需要本地化Text的组件，其所在物体都添加一个本地化器组件
  • 组件主要记录当前Text文本对应的键
  • 订阅了本地化管理器的语言改变事件，事件触发时控制刷新Text实际显示的文本。

本地化管理器：

  • 包含一个语言枚举类
  • 记录当前使用的语言
  • 维护了一个字典，键是Text文本键，值为当前语言的本地化文本，字典的数据从pojo类加载
  • 提供本地化方法，传入Text文本键，返回本地化后的文本
  • 提供一个语言变更事件，供本地化器订阅，语言变更时触发

成员

  • _instance，Instance：单例对象。
  • _currentLanguage：记录当前使用的语言
  • _localizationDict：存储本地化文本的键值对；键：文本标识，值：本地化文本
  • _locale：获取Java中的Locale类，用于获取安卓系统中的语言和地区
  • INSURANCE_LANG：保底语言，当系统语言不被支持或_currentLanguage被赋予异常值无法正常加载时，使用此语言。稍微增加那么一点鲁棒性。
  • INSURANCE_DICT：保底语言对应的本地化文本键值对
  • LOCALIZATION_FILE_PATH：存储本地化文件的路径前缀
  • OnLanguageChanged：语言改变事件。供本地化器订阅，当前语言改变时触发。
  • LanguageEnum：语言枚举类。这段LanguageEnum和MySystemLanguage是直接拿别人的。查了下基本上全部来自UnityEngine.SystemLanguage，不直接引用而自己另写一个，应该是便于修改，使用上更灵活吧。
  • MySystemLanguage：当前系统语言。
    • 如果是在Unity Editor中运行的话，直接从Application中获取系统语言。
    • 如果在Android系统中，则使用_locale获取系统语言编号，通过switch语句返回对应的枚举类。
    • switch写起来感觉有点麻烦，本质就是需要建立一个string到Enum的键值映射，有想过通过Enum.Parse来做，但Enum那边改动有点大，暂时没试。不过反正不是我写的，switch本质上也是O(1)的Hash，就先这样吧。
    • 这里的语言缩写，查了下应该使用的是ISO 639-1 Code 规范，可参考：ISO 639-1 Code。

方法

  • Awake：实例化时触发一次。
    • 单例赋值
    • 获取当前系统语言
    • 填充系统语言和保底语言的字典。先于各本地化器Start中的调用。
  • ChangeLanguage：更改语言方法。
    • 维护_currentLanguage变量
    • 重新加载当前语言字典
    • 触发语言改变事件。
    • 这里先判断了下当前语言和更改的目标语言是否是同一种，是的话就不做操作了；本意是提高效率，不加也行，考虑可能有强制reload的需求；同时考虑这两种场景的话，可以重载一下，在参数列表里加个bool变量来控制是否判断。
  • LoadDictionary：加载系统语言字典方法。
    • 如果途中除了什么问题的话，比如本地化文件找不到或格式有误，则加载保底语言的字典。
    • 如果保底字典也加载失败的话，就没什么办法了。
    • 加载json实际使用的是 UnityEngine.JsonUtility.FromJson(string json)
  • GenerateDict：字典转换方法。将pojo转Dictionary
  • GetString：获取本地化文本方法。
    • 首先在当前语言字典中寻找键对应的本地化文本，找到则返回
    • 如果没找到，则在保底语言字典中寻找
    • 如果在保底语言字典中也没有此键，则记录异常，直接返回键本身
  • GetLanguage和GetCountry：通过Java中的Locale类，获取当前系统语言和地区
