Unity纵向滚动文本组件

发表于 2026-03-23 分类于 Unity

前言

在很多 UI 场景里（比如金币变化、战力跳动、活动倒计时、积分榜刷新），直接替换文本会显得生硬。
在此讨论一种滚动文本组件的实现：用两个 TextMeshProUGUI 交替滚动，实现“数字滚动”视觉效果。

方案设计

思路

核心思路类似“双缓冲”：

  1. 准备两个文本组件 txtA 与 txtB；
  2. 当前显示组件从可视区中心位置移动到视野外；
  3. 另一个组件从视野外同步移动到可视区中心位置；
  4. 下一次滚动时交换二者角色，循环往复。

显示区域使用 RectMask2D，将超出窗口的内容裁掉，只保留中间可见区域，形成“窗口内滚动”的观感。

组件结构

父节点高度决定一次滚动位移，所以两个文本组件高度直接填充父节点高度即可，其他随意。

结构说明

组件对外暴露两个核心接口：

  • SetText：直接设置当前显示文本（无滚动）；
  • RollAnim：执行多段滚动，stringList的参数含义是“按顺序展示的文本序列”。

动画曲线

有两层曲线概念：

  • 整体序列曲线：控制整段动画的默认节奏（overallEasing）；
  • 末段单独曲线：最后一次滚动可使用不同曲线（lastRollEasing）。

将最后一次滚动的曲线单独拆出来，主要因为 RollAnim 通常播放的是“多段滚动拼接”的 Sequence。如果希望加入回弹效果，曲线中大于 1 的区间在序列里不一定能按预期生效。

虽然末段曲线仍会受到整体曲线影响，但综合实现复杂度，这个方案在当前场景下已经足够稳定。如果不需要回弹，lastRollEasing 直接留空即可，不会影响正常播放。
