Unity实现抓钩效果

发表于 2023-02-17 分类于 Unity

代码来源

B站视频：【Unity教程搬运】我创造了完美的抓钩… 你也可以！ UNITY 2D

源码：Grappling Hook

简介

视频演示的抓钩效果如下：

整个抓钩效果可大致分为两部分，一个是抓钩绳子的动画绘制，一个是物体的移动。

动画绘制用到了AnimationCurve和LineRenderer组件，第一次接触到，感觉挺有意思的，正好视频作者对绘制过程没有细说，在此就记录一下这部分的逻辑。

动画绘制，在Tutorial_GrapplingRope中。为简化说明，做了一个删去物体移动的版本，代码会和作者原版有一点小区别，但应该影响不大，完整版在文章末尾。

代码

鼠标交互

GetKeyDown方法，只在按键按下的第一帧执行，之后鼠标持续处于按下时不会执行。在这里为抓钩绳子的绘制做一些初始化操作。

GetKey方法，在按键持续按下时执行。在这里绘制抓钩的绳子。

GetKeyUp方法，在按键松开的一帧执行。做收尾的工作，禁用LineRenderer，隐藏抓钩的绳子。

绘制抓钩绳的初始化工作

  • startPos，物体当前的位置
  • endPos，当前鼠标的位置
  • direction，抓钩伸出的方向，即从物体位置指向向鼠标位置
  • waveSize，抓钩的实时宽度。在抓钩绳由松弛曲线逐渐紧绷直成为直线的过程，waveSize会不断减小直至为0。初始设置为最大宽度。
  • moveTime，绘制抓钩已用的时间。辅助计算抓钩运行到了什么位置
  • isStraight，当前抓钩绳是直线还是曲线。绳是由松弛曲线逐渐紧绷成为直线的，故初始设为false
  • percision，绘制绳子曲线时的精细度，即绘制曲线时，使用多少个点。

最后需要为LineRenderer的Position信息初始化。

绘制抓钩

在鼠标持续按下时会绘制抓钩的绳子，执行此方法。

  • 当isStraight为false
    • 当抓钩末端已到达终点，则将isStraight置为true，准备绘制直线
    • 当抓钩末端未到达终点，绘制曲线
  • 当isStraight为true
    • waveSize > 0，说明抓钩正在由松弛曲线绷成直线，让waveSize继续随时间减小。straightenLineSpeed 就是这个绷直过程的速度
    • waveSize <= 0，说明抓钩已经绷成直线，绘制直线

绘制直线

LineRenderer。就是个用来画线的组件。用 SetPosition 设置线中各点的位置，组件就能将其连成一条线画出来。

很简单，只设置两个点，分别处于抓钩的两端就行。开头对LineRenderer点数量的设置并非必要，只是做下优化，减少渲染的工作量。

绘制曲线

AnimationCurve。可以理解为设置一个曲线函数f(x)，调用AnimationCurve.Evaluate(x)方法时，返回的值就是f(x)。

这里用到了两个AnimationCurve，ropeAnimationCurve用于表现抓钩绳子的形状，ropeProgressionSpeed用于标识抓钩移动的速度特征。

首先根据 percision 将绳索的绘制分为各个小段，这里的delta可理解成ropeAnimationCurve曲线中的横坐标。

offset矢量是取抓钩伸出方向（即direction）的垂直方向，大小乘以ropeAnimationCurve曲线中对应delta的纵坐标。大小还乘了一个waveSize，可以手动控制曲线弯曲的幅度。

如果曲线x轴方向对应抓钩伸出的方向，则各个i值对应的offset表示如下，这里暂时没有考虑waveSize的影响。

Vector2.Lerp()线性插值。线性填充两点之间的空隙。

targetPosition指的是曲线上的每个点，从startPoint出发，运动的最终目的地是哪里。使用Vector2.Lerp()函数，正好取得direction上每一小段的矢量。和offset相加，成为目标地点。

currentPositoin 则是将 从startPosition开始，到targetPosition结束的矢量，利用线性插值分成各个小段，随时间推移，线上的点逐渐从startPosition移动到targetPosition。分段的紧密程度，也就是移动速度是先快后慢，还是先慢后快，还是匀速，由ropeProcessCurve速度特征曲线决定。这里使用的是直线，即为匀速。

currentPositoin的计算方式也许可以改进一下，当前曲线绘制的时间是固定的，和起始点的距离没关系。Evalue函数可以传(moveTime * ropeProgressionSpeed / direction.magnitude)。

这样一说，waveSize其实也可以和距离挂钩。在初始化的时候将waveSize设置为 maxWaveSize * direction.magnitude，此时maxWaveSize的含义就不是曲线最大宽度了，而是曲线最大宽度根据距离长短变化的系数。
