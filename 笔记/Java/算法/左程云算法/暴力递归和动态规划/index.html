暴力递归和动态规划

发表于 2022-04-09 分类于 算法

递归转换为动态规划

暴力递归就是尝试

  • 把所有问题转化为规模更小的同类子问题。
  • 有明确的不需要递归的子条件 base case。
  • 得到子问题的结果后，有明确的决策方案。
  • 不记录每个子问题的解。

尝试方式，尽量让单个可变参数维度低，尽量让可变参数数量少。
（比如int就是0维，int[]是1维）

暴力递归可以转换为动态对话，整个流程为：

  1. 从业务逻辑尝试，找出暴力递归的方法
  2. 转为记忆化搜索动态规划，不整理各个状态之间的依赖。
  3. 严格表结构的动态规划。（在某些问题上，记忆化搜索动态规划和严格表结构的动态规划有同样的复杂度）
  4. 根据严格表结构的动态规划进一步优化。

记忆化搜索动态规划转换为严格表结构的动态规划步骤：

  1. 分析可变参数的变化范围
  2. 标出计算的终止位置。
  3. 标出初始状态。（base case）
  4. 分析表格依赖
  5. 确定计算顺序，进行计算。

例：
给定整数N，表示有N个位置1~N，给定机器人的初始位置S和目标位置E，给定机器人必须走的步数K。
求出机器人从初始位置走到到目标位置，一共有几种行走路径。
机器人的行走规则，每次可以向左走一步（N到N-1），或者向右走一步（N到N+1）。在1位置只能向右走，在N位置只能向左走。

解：递归

用函数 f(N, E, rest, cur) ，表示N个位置，目标位置是e，剩余步数为rest，当前位置为cur时，有几种到达目标位置的路径。

则有

f(N, E, rest, cur) = f(N, E, rest - 1, cur - 1) + f(N, E, rest - 1, cur + 1); (cur != 1 && cur != N)
f(N, E, rest, cur) = f(N, E, rest - 1, cur - 1); (cur == N)
f(N, E, rest, cur) = f(N, E, rest - 1, cur + 1); (cur == 1)

base case则是

f(N, E, 0, cur) = 1; (cur == E)
f(N, E, 0, cur) = 0; (cur != E)

转换为记忆化搜索，加了一个缓存数组，避免递归过程中的重复计算。
例如f(N, E, 3, 1) 和 f(N, E, 3, 3)下一步都会调用 f(N, E, 2, 2) 造成重复计算。
此时f(N, E, 2, 2)的计算结果，和是谁调用的它，没有关系。称为无后效性。

斜率优化

根据dp表结构，和计算顺序。
dp[index][rest]的计算，需要依赖dp[index+1][rest]、dp[index+1][rest-1*arr[index]]、dp[index+1][rest-2*arr[index]]、…、dp[index+1][接近0]。
而在计算dp[index][rest-1*arr[index]]的时候，dp[index+1][rest-2*arr[index]]、dp[index+1][rest-3*arr[index]]、…、dp[index+1][接近0]的和值就已经计算过了。
所以dp数组计算方式可以优化为dp[index][rest] = dp[index+1][rest] + dp[index][rest-arr[index]]
