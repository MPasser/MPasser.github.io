Bean的生命周期

发表于 2022-06-24 分类于 Spring

概述

Bean的生命周期指Bean创建->初始化->销毁的过程。

可以自定义Bean初始化和销毁方法。

容器在Bean来到对应生命周期时，自动调用自定义的初始化或销毁方法。

BeanFactoryPostProcessor -> 代码块 -> 实例化 -> 数据装配 -> 初始化方法之前 -> 初始化方法 -> 初始化方法之后 -> 使用 -> 销毁方法 -> 在容器中销毁。

初始化和销毁

自定义Bean初始化和销毁方法。

使用XML配置时，可在bean标签中添加 init-method 或 destroy-method 属性，来设置自定义初始化或销毁方法。

在Bean注解中显式指定

指定初始化方法，init-method；指定销毁方法，destroy-method。

需要注意的是：对于非单例的Bean，容器只负责初始化，但不会管理Bean，容器关闭时，不会调用销毁方法。

在配置类中设置Bean为非单例。

此时连Bean的初始化都没有了，因为非单例的Bean是懒加载的，当使用的时候才会实例化。

BeanPostProcessor

Bean后处理器用来对bean的功能进行扩展增强，对IoC容器中的所有bean都有效。

包括Bean的赋值，注入其他组件，生命周期注解功能等。

Aware接口

adj.
意识到的;意识到;知道;明白;察觉到;发觉;发现;对…有兴趣的;有…意识的

实现了相应的Aware接口，这个类就获取了相应的资源。

比如实现了 ApplicationContextAware 接口，就获取了IOC容器 ApplicationContext；实现了 EnvironmentAware 接口，就获取了环境信息。
