Spring AOP

发表于 2022-06-29 分类于 Spring

概述

AOP：Aspect Oriented Programming 面向切面编程。

将程序中交叉业务逻辑（事务、日志）代码提取出来，封装成切面，由AOP容器在适当时机（位置）将封装的切面动态地织入到具体业务逻辑中。

适用于具有横切逻辑的场合，如事务管理、日志记录、性能监测、异常通知、访问控制等。

Spring AOP原理就是使用动态代理。如果被代理类实现了接口，则使用Proxy类实现；如果没有实现接口，则通过CGLIB继承来实现。

通知方法

  • 前置通知：@Before ，在目标方法运行之前执行。
  • 后置通知：@After ，在目标方法运行之后执行，无论方法是正常还是异常结束。
  • 返回通知：@AfterReturning ，在目标方法正常返回后执行。
  • 异常通知：@AfterThrowing ，在目标方法出现异常后执行。
  • 环绕通知：@Around ，动态代理，最底层通知，手动指定执行目标方法。

注解的一些回调属性

注解有一些预定义的属性，用于获取相应的信息。

例如 @AfterThrowing throwing ，用于获取切面目标方法所抛出的异常。

在切面方法中，在参数中添加对应变量，将变量名注册到注解的 throwing 属性中，在方法中就可以获取切面目标方法的参数列表。

这些注解都有 argNames 属性，用于获取目标方法的参数。

使用时，需要在切入点定义中指明目标方法的参数列表，argNames属性和切面方法的参数列表都需要和这里指明的相同。

连接点

JoinPoint。在切面方法中定义切入点，可以获取更多目标方法中的信息。

AOP相关类的创建流程

之前在配置类中，声明了注解 @EnableAspectJAutoProxy ，查看注解的定义，引入了 AspectJAutoProxyRegistrar.class 组件

而这里的 Registrar 实现了 ImportBeanDefinitionRegistrar 接口，在接口方法中主要注册了 AnnotationAwareAspectJAutoProxyCreator.class 这个类。

AnnotationAwareAspectJAutoProxyCreator 类的关系图如下，

AOP核心就是创建 AnnotationAwareAspectJAutoProxyCreator 对象。

该类最终实现了接口 BeanPostProcessor （后置处理器）。创建后置处理器的对象放在容器中。
（最终创建的BeanID为AUTO_PROXY_CREATOR_BEAN_NAME = “org.springframework.aop.config.internalAutoProxyCreator”）
（其实所有注解都有对应的后置处理器）。

其他Bean组件创建对象的时候，都会被 AnnotationAwareAspectJAutoProxyCreator 拦截，影响其创建过程。

普通单例Bean的创建是在 容器执行 finishBeanFactoryInitialization 方法时。但如果是一个实现了 BeanPostProcessor 接口的Bean，则其创建的过程会更早发生，在执行 registerBeanPostProcessors 方法时就会创建。

AOP调用流程

执行被增强类对象的方法时，代理对象就会调用之前保存好的切面通知，代码会跳转到代理类（此处是CglibAopProxy类）的 intercept 方法。

此方法主要做了两件事情：

  1. 获取拦截链（MethodInterceptor）
  2. 链式调用通知方法

其中多个方法递归调用的顺序和执行顺序是一致的：

  1. Around
  2. Before
  3. After
  4. AfterReturning
  5. AfterThrowing

拦截链执行的顺序没有受递归影响而倒置，看上去是因为用了

另外，使用环绕通知时，需要手动调用proceed方法，这里proceed实际就是CglibMethodInvocation类的，也就是参与到通知方法递归调用proceed的过程中了。
