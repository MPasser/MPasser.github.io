发表于 2022-07-24 分类于 Spring

概述

常用的Web三大组件如下：

  • servlet。运行在服务器端的小程序，一个接口，定义了Java类被浏览器访问到的规则。
  • filter，过滤器。当访问服务器的资源时，过滤器可以将请求拦截下来，完成一些特殊的功能。
  • listener，事件监听机制。
    • 事件：一件事情。
    • 事件源：事件发生的地方。
    • 监听器 ：一个对象。
    • 注册监听：将事件、事件源、监听器绑定在一起。当事件源上发生某个事件后，执行监听器代码。

经常在web.xml中配置使用。

Servlet的异步处理

Tomcat主线程池获取连接请求响应等信息，新建线程，异步对这些请求进行具体业务逻辑的处理。Tomcat的主线程不被阻塞直接返回，释放Tomcat连接。

虽然是异步，但在当前例子中，用户体验区别并不大，访问对应地址后都需要等待3000ms才能看到结果，此前都是白屏。

但异步的好处在于将 接收请求 和 业务操作 的工作分开了。主线程接收完请求，将具体业务交给另一个线程后，自己就解放了，可以再接收其他的请求。

有一点值得注意，这里只访问了对应地址一次，但之前写的拦截器，在此处被触发了两次。

根据官方文档的描述，其执行过程如下：

  • Controller 返回一个 Callable对象。
  • SpringMVC 调用 request.startAsync() 并将 Callable 提交给 TaskExecutor ，在另一个线程异步处理。
  • 同时，DispatcherServlet 和所有的过滤器退出Servlet容器的线程，但response保留开放。
  • 最终 Callable 会生成一个结果，且SpringMVC将请求分发回Servlet容器，以完成整个流程。
  • DispatcherServlet 再次被触发，流程获取 Callable 对象的返回值，恢复运行。

也就是说，DispatcherServlet 被触发了两次，所以能看到拦截器输出了两次，这两次可能是不同的线程在执行。

另外需要注意的一点是，这里返回的 Callable 对象，需要放在一个线程池里处理，没有指定的话，Spring会自动指定一 SimpleAsyncTaskExecutor 对象。

这个线程池每来一个任务就新建一个线程，也不会重复利用之前的线程，所以不推荐，会报Warning。

推荐的做法是在配置类中指定执行异步请求时使用的线程池。
