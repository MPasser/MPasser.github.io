BeanFactory的后置处理器

发表于 2022-07-17 分类于 Spring

概述

补充解释两个Spring组件。

  • BeanFactoryPostProcessor：BeanFactory的后置处理器
  • BeanDefinitionRegistryPostProcessor：Bean定义的后置处理器

BeanFactoryPostProcessor

预先处理 BeanFactory。

回顾Bean的生命周期：

BeanFactoryPostProcessor -> 代码块 -> 实例化 -> 数据装配 -> 初始化方法之前 -> 初始化方法 -> 初始化方法之后 -> 使用 -> 销毁方法 -> 在容器中销毁。

BeanFactoryPostProcessor是在所有Bean实例化前就执行了；而普通BeanPostProcessor是在每个Bean实例化后，执行初始化前后执行。

BeanDefinitionRegistryPostProcessor

预先处理 定义Bean 的 注册器。

BeanDefinitionRegistryPostProcessor 是 BeanFactoryPostProcessor 的子接口。所以实现类需要实现 BeanDefinitionRegistryPostProcessor 的 postProcessBeanDefinitionRegistry 方法 和 BeanFactoryPostProcessor 的 postProcessBeanFactory 方法。

BeanFactoryPostProcessor 的作用是在Bean的定义信息已经加载但还未创建初始化时，执行 postProcessBeanFactory 方法。

而 BeanDefinitionRegistryPostProcessor 的执行时机更早。
