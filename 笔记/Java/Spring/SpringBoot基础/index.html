SpringBoot基础

发表于 2022-08-02 分类于 Spring

概述

SpringBoot提供了一种快速使用Spring的方式，基于约定优于配置的思想，可让开发人员不必在配置与逻辑业务之间进行思维的转换，提高开发效率。

2014年4月，SpringBoot 1.0.0发布。Spring的顶级项目之一。官网

Spring的缺点

配置繁琐

虽然Spring的组件代码是轻量级的，但它的配置却是重量级的。从XML配置到引入注解扫描，再到基于Java的配置，都不算很简洁。

SpringBoot的自动配置是一个运行时（更准确的说，是应用程序启动时）的过程，该过程是SpringBoot自动完成的。

依赖繁琐

环境搭建时，需要分析要导入哪些库的坐标。不同的坐标需要处理好版本间关系，和所依赖包之间的冲突问题。

SpringBoot使用的是起步依赖，本质上是一个Maven项目对象模型（POM ，roject Object Model），定义了对其他库的传递依赖，这些东西加在一起即支持某项功能。

简单地说，起步依赖就是将具备某种功能的坐标打包到一起，并提供一些默认的功能。（我理解就是针对场景给你提前组合好了）

此外，SpringBoot还提供了一些辅助功能，如嵌入式服务、安全、指标、健康监测和外部配置等。

SpringBoot并不是对Spring功能上的增强，而是提供了一种快速使用Spring的方式。

需要注意的是，SpringBoot创建项目时，使用的是jar的打包方式。

  • 在各种 starter 中，定义了完成该功能需要的坐标合集，大部分信息来自于父工程。所以版本号不用写，会从父工程继承。
  • 工程继承parent，引入starter后，通过依赖传递，就可以简单方便获得所需jar包，并不会存在版本冲突等问题。

内部配置加载顺序

SpringBoot程序启动时，加载配置文件的优先级从高到低为：

  1. file:./config/ ，当前项目中的/config 目录下
  2. file:./ ，当前项目的根目录下
  3. CLASSPATH:/config/ ，classpath的config目录
  4. CLASSPATH:/ ，classpath的根目录

外部配置加载顺序

常用外部配置的优先级从高到低为：

  1. 命令行参数
  2. 命令行指定的properties文件
  3. jar包目录下，config目录下的properties文件
  4. jar包同目录下的properties文件

更多其他配置方式的优先级可查阅官方文档

监听机制

SpringBoot的监听机制，其实是对Java提供的事件监听机制的封装。

Java中的事件监听机制定义了一下几个角色：

  • 事件：Event，继承 java.util.EventObject 类的对象。
  • 事件源：Source，任意对象Object。
  • 监听器：Listener，实现 java.util.EventListener 接口的对象。

SpringBoot在项目启动时，会对几个监听器进行回调，可以实现这些监听器接口，在项目启动时完成一些操作。

  • ApplicationContextInitializer
  • SpringApplicationRunListener
  • CommandLineRunner
  • ApplicationRunner

Spring Boot Admin

是一个开源社区项目，用于管理和监控SpringBoot应用程序。Spring Boot Admin有两个角色，客户端（Client）和服务端（Server）。应用程序作为Client 向 Server 注册； Server 提供UI界面展示 Client 上Actuator Endpoint上的监控信息。

Server端需要注意需要在引导类上加 @EnableAdminServer 注解启用监控功能；Client端需要注意在配置文件中指定Server的url 。
