线程的并发工具类

发表于 2022-05-08 分类于 多线程

Fork/Join

分而治之

使用分而治之的思想。

在必要的情况下，将一个大任务，拆分成若干的小任务，直至问题规模小到不需要再拆分，再将所有小任务的运算结果进行join汇总。
拆分出来的子任务，与原任务问题形式相同。

工作密取 Work Stealing

提高CPU利用率，尽量不要有空闲的线程。

线程A的任务执行完毕，但任务B还剩下几个任务，此时线程A会将B剩下的任务偷过来执行，之后将执行完的结果放回线程B中。
从线程B任务队列的尾部偷取。

使用的标准范式

  1. pool = new ForkJoinPool();
  2. MyTask myTask = new ForkJoinTask();
  3. Pool.invoke(myTask);
  4. Result = myTask.join();

问：ForkJoinPool的作用是什么，为什么一定要使用？或者说应该如何理解ForkJoinPool在这里担任的角色。

答：ForkJoinPool是一个特殊的线程池，专门用来启动Fork/Join任务的。池中的线程数是由CPU核数决定的。

Fork/Join的异步使用

需要注意的是，在上节例子中，main方法里使用tester.join();启动Fork/Join任务，这样会导致main方法阻塞，等到Fork/Join任务完成之后才能继续运行。
如果想让Fork/Join任务异步运行，可以使用ForkJoinPool.submit()，使用Future类存储任务结果，调用future.get()的时候才会阻塞。

invokeAll(subTask) 这里，参数要传多个 RecursiveAction 实例，也可以传集合类。

问：ForkJoinPool 有 submit()、invoke()、execute()方法，都有什么区别？

CountDownLatch

用来控制一个（或多个）线程等待多个线程，可以看做加强版join()。
await()方法用于阻塞等待，countDown()方法用于计数器减一。

从结果也可以看出，getCount()方法也不是线程安全的。

CyclicBarrier

控制多个线程互相等待，只有多个线程都到达await()时，这些线程才会继续执行，也是维护计数器实现的。
可以用于比如多个表单计算结果，等待所有表单计算完成后汇总结果这种场景。

可以看出，getNumberWaiting()方法也不是线程安全的。

CountDownLatch放行由第三者控制，CyclicBarrier放行由一组线程本身控制。

Semaphore

控制对互斥资源的访问线程数。常用与流量控制。

问：这里为什么要使用两个Semaphore？

答：用两个的意义在于，限制 同时 获取连接的线程 和 归还连接的线程 的总和，不超过5。只设置used的话，同时归还连接的线程数就不受控制了。

Exchanger<V>

用于两个线程之间的数据交换。
线程A第一次调用exchange(V val)时，会准备好V类型的值在此阻塞等待；线程B此时调用exchange方法时，也准备好V类型的值；两个线程交换数据后，继续向下执行。

Callable、Future和FutureTask

Runnable没有返回值，且无法抛出异常；
Callable有返回值，可以抛出异常。

RunnableFuture接口，继承Runnable接口和Future接口；
FutureTask是RunnableFuture的一个实现类，同时有Runnable和Future的特性。

FutureTask实例化的时候，可以传Callable或者Runnable对象进行构造。传Runnable对象的时候，还需要另外传一个变量来接收任务执行结果。

  • isDone()：判断任务是否已结束，无论是正常结束或异常中断，都返回true。
  • isCancelled()：若任务完成前被取消，返回true。
  • cancel(boolean)：任务还未开始或已经结束，返回false；任务正在运行，调用cancel(true)，会传一个interrupt信号，若中断成功返回true；调用cancel(false)则不会中断。

问：cancel(false)有什么用？

cancel(false)用于取消还未开始的任务，cancel(true)用于取消所有状态的任务。当FutureTask处于已启动状态时，执行FutureTask.cancel(false)方法将不会对正在执行此任务的线程产生影响（让正在执行的任务运行完成）;但是当FutureTask处于未启动状态时，执行FutureTask.cancel()方法将导致此任务永远不会被执行。

Thread包FutureTask，FutureTask包Callable
