Hexo项目搭建

发表于 2020-09-10 分类于 Web ， Hexo

前言

记录搭建Hexo时的大体步骤及遇到的一些问题，留给自己作参考。

Hexo是一个博客框架，点击此处访问官网。文章的未覆盖、过时或错误内容，可尝试从官方文档找答案。

环境准备

Git

Git，开源的分布式版本管理工具。

Hexo自身需要版本管理，有些软件内容也是通过git分发的。

直接yum安装。

Nginx

Nginx，(engine x) 是一个高性能的HTTP和反向代理Web服务器。

安装

直接yum安装。

配置

默认配置文件位置在 $NGINX_HOME/nginx.conf 。而 $NGINX_HOME 在Linux用yum安装后，默认值为 /etc/nginx

配置文件的结构可参考官方教程或官方文档。

主要需要修改的配置就一个，服务器根目录的位置；因为Hexo生成静态页面的主页路径为 $HEXO_HOME/public/index.html，所以根目录位置改为 $HEXO_HOME/public 就行。

启动与关闭

使用 nginx 命令启动服务器，启动后，可以通过使用-s参数调用可执行文件来控制

signal可取：

  • stop — fast shutdown
  • quit — graceful shutdown
  • reload — reloading the configuration file
  • reopen — reopening the log files

NVM

NodeJS

NodeJS 是一个基于Chrome V8引擎的JavaScript运行时。简单来说，服务器安装了NodeJS之后，能够让在网页上运行的JavaScript也能在服务器端运行。即安装完NodeJS后就能在服务器使用JavaScript写脚本了。

而hexo是依赖于NodeJS的，所以需要安装NodeJS。

NVM

如果说手上有好几个项目，每个项目的需求不同，进而不同项目必须依赖不同版的NodeJS运行环境。如果没有一个合适的工具，这个问题将非常棘手。

NVM （Node Version Manager）是一个独立于NodeJS的外部shell脚本，能够帮助管理NodeJS的版本。项目使用git下载，随便找个位置装一下。

来拷贝它。

之后需要执行nvm文件夹下的nvm.sh脚本：source nvm.sh来初始化，之后便可使用nvm或npm相关命令了。

建议把此初始化命令source nvm.sh加到~/.bashrc文件里，这样每次登录后就能自动初始化了。

使用nvm ls查看已安装的NodeJS版本，使用nvm install [node_version]来安装版本号为[node_version]的NodeJS。具体安装什么版本可查看Hexo文档中写的安装前提，或直接 nvm install latest安装最新版本。

NPM

NPM（Node Package Management）是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题，常见的使用场景有以下几种：

  • 允许用户从NPM服务器下载别人编写的第三方包到本地使用，hexo就是这样一个第三方包。
  • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

详细介绍可见：NPM 使用介绍

Hexo

Hexo是一个快速、简洁且高效的前端博客框架，用于生成静态页面。对markdown的支持还不错。

使用 npm install hexo-cli -g 安装。

如果用默认的npm源，安装依赖超时的话，可换用国内阿里巴巴的npm源

初始化

  1. 创建一个文件夹作hexo项目的根目录，如 hexo_site，进入文件夹，使用命令 hexo init 来初始化一个hexo项目。

  2. 使用 hexo generate 命令生成静态页面，位置在该项目的.../hexo_site/public文件夹中。

  3. 如前述，将Nginx配置文件中的root变量配置为上述的.../hexo_site/public文件夹位置。

  4. 博文位置在hexo项目的.../hexo_site/source/_post文件夹。

配置与问题

站点基本信息配置

hexo项目的配置文件在项目根目录的 .../hexo_site/_config.yml 文件中，具体配置可参考官方文档。

语言

语言在项目根目录的配置文件 _config.yml 设置 language 项。

使用不同的主题可能需要设置成不同的值，需参考主题的文档相应设置，对于NexT主题，虽然文档中写的中文需要设置成zh-Hans，但实际需要设置值为zh-CN。之后不知道官方文档会不会更新，可以直接用下面的方法找正确的配置值。

站点使用的语言，实际使用的是主题文件夹中的language文件夹.../hexo_site/theme/language 。里面有什么yml文件就说明支持哪些语言，配置的值对应文件名。配置成不支持的语言，一般就使用此文件夹中的第一个文件。

图标

网站图标在主题设置中，对于NexT主题，配置文件在.../hexo_site/theme/next/_config.yml ，配置的条目为favicon，默认图片资源的路径在.../hexo_site/theme/next/source/image/中，如果需要自定义图标，建议图片也放在这里。

标签或分类页面无法访问

这两个页面需要手动创建。在项目根目录下使用 hexo new page tags 命令创建标签页面。页面生成在项目的 .../hexo_site/source/tags/index.md 。

编辑此文件头，title的值即为访问标签时的页面标题。

分类同理，用的命令是 hexo new page categories。
