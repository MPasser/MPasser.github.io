Ajax笔记

发表于 2020-07-25 分类于 Web

简介

Asynchronous Javascript And XML (异步 JavaScript 和 XML)，只刷新局部页面的技术

Ajax引擎：即XMLHTTPRequest对象，以异步方式在客户端和服务器之间传递数据

为什么使用Ajax

  • 无刷新：不刷新整个页面，只刷新局部
  • 只更新部分页面，有效利用带宽
  • 提供连续的用户体验
  • 提供类似于C/S的交互效果，操作更方便

传统Web与Ajax的差异

差异 方式 说明
发送请求不同 传统web
Ajax		 提交表单方式发送请求
异步引擎对象发送请求
服务器响应不同 传统web
Ajax		 响应内容是一个完整页面
响应内容只是需要的页面
客户端处理方式不同 传统web
Ajax		 需等待服务器响应完成并重新加载整个页面后用户才能进行操作
可以动态更新页面中的部分内容，用户不需要等待请求的响应

Ajax工作流程

XMLHttpRequest对象是整个Ajax技术的核心，它提供了异步发送请求的能力

常用方法

  • open(method, URL, async)
    • 建立与服务器的连接
    • method参数：请求方式，可选值是GET（默认）或 POST
    • URL：请求地址
    • async参数：是否使用异步请求，可选值是true（默认）或 false
  • send(content)
    • 发送请求
    • content：请求参数，只有post请求才能通过send()方法发送参数
  • setRequestHeader(header, value)
    • 设置请求头信息

常用事件

onreadystatechange：事件属性，指定回调函数，当readyState属性改变时触发执行

常用属性

readyState：表示XMLHttpRequest对象的状态

就绪状态码 说明
0 XMLHttpRequest对象已创建，尚未初始化
1 对象开始发送请求
2 对象请求发送完成
3 对象开始读取响应，尚未读取完成
4 对象读取响应结束

status：表示HTTP请求响应的状态

状态码 说明
200 服务器响应正常
400 无法找到请求的资源
403 没有访问权限
404 访问的资源不存在
500 服务器内部错误

responseText：获得响应的文本内容

responseXML：获得响应的XML文档对象Document

传统方式实现Ajax的不足

  • 方法、属性、常用值较多，不好记忆
  • 步骤繁琐
  • 浏览器兼容性问题

jQuery常用Ajax方法

方法 作用
$.ajax() 发送HTTP请求加载远程数据，最底层的Ajax实现
$.get() 发送HTTP GET请求加载数据
$.post() 发送HTTP POST请求加载数据
$.getJSON() 发送HTTP请求从服务器加载JSON数据
load() 发送Ajax请求从服务器加载数据，并把返回的数据放置到指定的元素中，最简单的方法

$.ajax([settings])的参数属性说明

属性类型和名称 说明
String type 请求方式（POST或GET，默认为GET）
String url 发送请求的地址，默认为当前页地址
boolean async true为同步，false为异步，默认为异步
Object data 或 String data 发送的数据类型，包括：XML、HTML、Script、JSON、JSONP、text（默认）
Number timeout 设置请求超时时间
boolean global 表示是否触发全局Ajax事件，默认为true
processData
contentType

常用函数参数

函数 说明
function success(Object result, String ts) 请求成功后调用的函数
result：可选，由服务器返回的数据
ts：可选，type stirng ,描述请求类型的字符串
function error(XMLHttpRequest xhr, String em,Exception e) 请求失败时调用的函数
xhr：可选，请求失败的xhr对象
em：可选，error message ，错误信息
e：可选，捕获的异常对象
function beforeSend(XMLHttpRequest xhr) 发送请求前调用的函数
xhr：可选，发送请求的xhr对象
function complete(XMLHttpRequest xhr, String ts) 请求完成后调用的函数，不论成功与否
xhr：可选，发送请求的xhr对象
ts：可选，type stirng ,描述请求类型的字符串

Ajax跨域请求

一个完整的URL包括8个部分（protocol、host、port、path、resource、query、string、anchor、auth），以上部分若protocol、host和port中有任何一个不同就是跨域
浏览器的同源策略限制XMLHttpRequest向另外一个网站发送请求，即Ajax是无法直接跨域的
跨域的解决方式：

  • JSONP
  • CORS
  • iframe

JSONP（JSON with Padding）

JSONP(JSON with Padding) 是 json 的一种”使用模式”，可以让网页从别的域名（网站）那获取资料，即跨域读取数据

JSONP只支持GET请求方式，即使使用POST方式，也会自动被转换为GET方式

一般的数据api都会支持jsonp的调用方式，此时需要在请求参数中指定返回数据格式为jsonp，并指定回调callback的名字

CORS（Cross-Origin Resource Sharing）

CORS是一种允许当前域（domain）的资源（比如html/js/web service）被其他域（domain）的脚本请求访问的机制

在服务器算允许跨域请求
